¿Qué es la Programación Orientada a Objetos?

La Programación Orientada a Objetos (POO) es un paradigma de programación basado en el concepto de "objetos", que encapsulan datos y comportamientos.

Asi mismo nos permite organizar el código en unidades lógicas y reutilizabes. Ademas ayuda a pensar en objetos en lugar de la lógica del programa

Tambien permite crear soluciones eficientes y previene el acceso no autorizado a los datos.

1. Definición de Clases

Las Clases son plantillas que definen Atributos y el comportamiento de los objetos

Tambien son modelos o abstracciones de la realidad, definen las variables, las propiedades, procedimientos y eventos de los objetos.

Ademas son como moldes o plantillas que contienen la definicion y eventos de entidades del mundo real, y son grupos de objetos similares.

Definición de Objetos

Los objetos son instancias de esas clases.

Tambien son copias creadas a partir de una clase, ademas tienen los comportamientos de esas clases.

Tambien son instancias del mundo real, como un boligrafo, un computador portatil,etc. Tambien son llamadas instancias de clases

                
// Ejemplo en Python
class Persona:
    def __init__(self, nombre, edad):
        self.nombre = nombre
        self.edad = edad

    def saludar(self):
        print(f"Hola, mi nombre es {self.nombre} y tengo {self.edad} años.")

2. Definicion de Herencia

La herencia es un mecanismo que permite a una clase heredar propiedades y comporamientos de otra.

Herencia Simple: Se define una nueva clase a partir de la clase incial.

Herencia Multiple: Se define una nueva clase a partir de dos o mas clases iniciales.

Defincion de Encapsulación

El encapsulamiento consiste en agrupar datos y metodos relacionados en una clase, y controlar el acceso a ellos.

Con el encapsulamiento logramos proteger la integridad de los datos, ademas ocultar los valores o estados de un objeto.

Tambien evitamos el acceso directo a los datos y garatizar que a los datos solo se accedan a travez de servicios propuestos.

                
// Ejemplo en Python
class Estudiante(Persona):
    def __init__(self, nombre, edad, carrera):
        super().__init__(nombre, edad)
        self.carrera = carrera

3. Polimorfismo

El polimorfismo es la capacidad de que distintos objetos respondan de manera diferente a un mensaje.

.Esto permite que se incorporen nuevas funciones y comportamientos sin alterar el código existente.

Asi mismo el polimorfismo permite crear código mas flexible y reutilizable, permite diseñar clases que se puedan adaptar a diferentes situaciones.

Nos permite escribir codigo mas eficiente y nos permite redefinir métodospara clases derivadas.

                
// Ejemplo en Python
class Ave:
    def volar(self):
        print("Esta ave puede volar.")

class Pinguino(Ave):
    def volar(self):
        print("Los pingüinos no pueden volar.")
